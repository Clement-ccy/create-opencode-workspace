# 学术论文写作语调

## 概述
本技能定义学术论文的正式写作风格与章节结构模板。用于 `/write` 任务，面向期刊与会议论文。

## 写作风格原则
- **正式、精确、客观**：避免情绪化或口语化表达。
- **证据驱动**：每个结论必须有数据或文献支持。
- **术语一致**：同一概念使用同一术语与符号。

## 语言约定
### 主动 vs 被动
- 方法与流程可用被动语态强调客观性。
- 贡献与发现可用主动语态强调主体。

### 谨慎表述 (Hedging)
| 不可用 | 推荐用法 |
|---|---|
| “证明了” | “表明/暗示/支持” |
| “彻底解决” | “在…条件下有效” |
| “最佳” | “优于…基线” |

### 引用嵌入
- **Markdown**：`[Author et al., Year]`
- **LaTeX**：`\cite{key}` / `\citep{key}` / `\citet{key}`

## 章节结构模板
可以参考 `\references\*` 中的模板
### Abstract 模板
```text
背景 → 问题 → 方法 → 结果 → 影响
```

### Introduction 模板
```text
领域背景 → 研究空白 → 本文方法概述 → 贡献列表 → 论文结构
```

### Related Work 组织策略
- **按主题**：方法类别或问题维度。
- **按时间**：用于快速演化领域。
- **按对比**：与本文最相关工作优先。

### Method 规范
- 统一符号：首次出现需定义。
- 给出算法流程或伪代码。
- 指明复杂度或资源需求。

### Results 展示
1. 结果表格/图 → 解释变化趋势。
2. 关键发现 → 回答研究问题。
3. 异常结果 → 合理分析。

### Discussion 结构
影响/意义 → 局限 → 未来工作。

## Markdown vs LaTeX 使用规则
| 场景 | Markdown | LaTeX |
|---|---|---|
| 快速草稿 | ✅ | ❌ |
| 正式投稿 | ⚠️ 视会议要求 | ✅ |
| 公式密集 | ⚠️ | ✅ |

### 表格示例
```markdown
| 方法 | 准确率 | F1 |
|---|---|---|
| Ours | **91.2** | **89.7** |
| Baseline | 88.5 | 86.3 |
```

```latex
\begin{tabular}{lcc}
\toprule
Method & Acc & F1 \\
\midrule
Ours & \textbf{91.2} & \textbf{89.7} \\
Baseline & 88.5 & 86.3 \\
\bottomrule
\end{tabular}
```

## 常见反模式
- **过度主张**：避免绝对词。
- **贡献不清**：贡献必须可验证。
- **消融缺失**：关键组件需消融。
- **符号不一致**：同一量不要多个符号。

## 各章节检查清单
### Abstract
- [ ] 包含问题与结果
- [ ] 结果为量化指标

### Introduction
- [ ] 明确研究空白
- [ ] 贡献列表清晰

### Method
- [ ] 符号定义完整
- [ ] 算法流程清楚

### Results
- [ ] 表格/图有解释
- [ ] 统计显著性说明

### Discussion
- [ ] 局限与未来工作明确
