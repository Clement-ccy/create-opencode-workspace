# AI/机器学习研究领域

## 概述
本技能用于 AI/ML 研究任务的领域上下文与评估规范。适用于深度学习、强化学习、生成式模型、优化算法等方向的实验设计与论文写作。

## 研究方向与关注点
- **深度学习 (Deep Learning)**：架构创新、训练稳定性、泛化与可解释性。
- **强化学习 (RL)**：策略优化、样本效率、探索-利用权衡。
- **生成式模型 (Generative Models)**：分布拟合、样本质量、多样性与控制性。
- **优化与学习理论 (Optimization)**：收敛性、复杂度、优化器行为。

## 常见实验模式
1. **训练循环 (Training Loop)**：固定训练步数或早停，记录 loss/metric 曲线。
2. **超参搜索 (HPO)**：网格/随机/贝叶斯搜索，报告搜索空间与预算。
3. **消融实验 (Ablation)**：逐项移除或替换组件，说明动机与对照。
4. **规模扩展 (Scaling)**：模型大小、数据规模、计算预算的系统性变化。

```text
典型实验流水线
数据预处理 → 训练 → 验证/调参 → 测试 → 误差分析 → 可复现实验
```

## 标准指标 (按任务)
| 任务类型 | 常见指标 | 说明 |
|---|---|---|
| 分类 | Accuracy / F1 / AUC | 类别不均衡时优先 F1/AUC |
| 生成 | BLEU / ROUGE / BERTScore | 文本生成常用 |
| 语言建模 | Perplexity | 需说明 tokenization |
| 图像生成 | FID / IS | FID 更稳定 |
| 强化学习 | Average Return / Success Rate | 需报告方差 |

## 标准数据集（按子领域）
| 子领域 | 典型数据集 | 备注 |
|---|---|---|
| 视觉 | CIFAR-10/100, ImageNet | 需说明预处理与分辨率 |
| NLP | GLUE, SQuAD, WMT | 需说明评测脚本版本 |
| 生成 | CelebA, LAION | 注意版权与过滤策略 |
| RL | Atari, MuJoCo, Procgen | 需固定 seed 与环境版本 |

## ML 论文结构约定 (NeurIPS/ICML/ICLR)
1. **Abstract**：问题→方法→结果→贡献。
2. **Introduction**：研究背景、现有不足、本文贡献列表。
3. **Related Work**：按主题或方法分组，避免时间流水账。
4. **Method**：形式化定义、算法流程、复杂度分析。
5. **Experiments**：数据集、基线、设置、指标、结果与消融。
6. **Conclusion**：总结与局限。

## 常见对比基线
- **SOTA 方法**：最近 1-2 年同任务最强模型。
- **经典基线**：如 ResNet、Transformer、PPO/TD3 等。
- **简化版本**：去掉关键组件的自身基线。

## 可复现性要求
- 固定 **随机种子**，报告每次实验 seed。
- 报告 **硬件规格**（GPU/CPU/内存）。
- 报告 **训练细节**（batch size、lr、scheduler）。
- 结果必须包含 **均值±标准差**，必要时提供置信区间。

## 报告规范
- 明确 **训练/验证/测试划分** 与数据泄漏防护。
- 表格标注最优值（**加粗**）并给出方差。
- 对失败案例给出分析，不只展示成功结果。
