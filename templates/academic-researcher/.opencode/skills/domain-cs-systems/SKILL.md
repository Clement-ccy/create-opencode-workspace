# 计算机系统研究领域

## 概述
本技能用于系统研究方向的实验设计、评估方法与论文规范。覆盖分布式、操作系统、网络、数据库、编译器等方向。

## 研究方向
- **分布式系统**：一致性、容错、可扩展性。
- **操作系统**：调度、内存管理、系统调用优化。
- **网络**：拥塞控制、路由、协议设计。
- **数据库**：事务处理、索引、查询优化。
- **编译器**：优化 passes、IR 设计、代码生成。

## 常见实验模式
1. **基准测试 (Benchmarking)**：标准基准 + 自定义 workload。
2. **延迟/吞吐测量**：p50/p95/p99，QPS/OPS。
3. **扩展性测试**：节点数/核心数/数据规模逐步扩展。
4. **公平对比**：同硬件、同配置、同 workload。

```text
系统评测最小闭环
配置固定 → 热身 → 负载测试 → 统计分布 → 对比分析
```

## 标准指标
| 指标 | 说明 | 典型场景 |
|---|---|---|
| Throughput | 系统处理能力 | 数据库/网络 |
| Latency (p50/p95/p99) | 尾延迟 | 分布式/网络 |
| QPS/OPS | 查询/操作吞吐 | 服务端 |
| Memory/CPU | 资源使用 | OS/编译器 |

## 常见基准 (按子领域)
| 子领域 | 典型基准 | 说明 |
|---|---|---|
| 数据库 | YCSB, TPC-C, TPC-H | 事务与分析型 |
| 编译器 | SPEC, LLVM test-suite | 代码性能 |
| 网络 | iperf, RFC 测试 | 带宽/延迟 |
| 分布式 | Jepsen, Microbenchmarks | 一致性与吞吐 |

## 系统论文结构约定 (OSDI/SOSP/NSDI/SIGCOMM)
1. **问题与动机**：系统痛点 + 可测量目标。
2. **设计与实现**：架构图 + 关键组件。
3. **评估**：与基线对比、扩展性、灵敏度分析。
4. **讨论**：局限性与适用场景。

## 评估方法学
- **热身 (Warm-up)**：避免冷启动影响。
- **重复运行**：至少 5 次，报告均值与方差。
- **公平对比**：同硬件/同配置/同负载。
- **尾延迟**：必须报告 p99。

## 子领域 → 基准 → 指标速查表
| 子领域 | 典型基准 | 典型指标 |
|---|---|---|
| 数据库 | YCSB/TPC-C | Throughput, p95/p99 |
| 分布式系统 | Microbench/Jepsen | 延迟分布, 可用性 |
| 网络 | iperf/RFC | 带宽, RTT |
| 编译器 | SPEC | 运行时间, 代码大小 |
| 操作系统 | lmbench | syscall 延迟, 吞吐 |

## 报告规范
- 明确配置（CPU、内存、网络、OS 版本）。
- 结果图必须标明单位与误差条。
- 解释异常值与退化场景。
