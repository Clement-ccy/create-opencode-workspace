# 播客规范

## 平台特点

- **受众**：通勤/运动时收听，偏好深度内容
- **形式**：音频为主，可视频播客
- **时长**：15-60 分钟
- **特点**：陪伴感、亲密感、适合延伸讨论

---

## 内容类型

### 1. 独白/分享

**核心**：一个人讲述观点或经历

**适合**：
- 个人随笔延伸
- 行业观察
- 思考分享

### 2. 访谈

**核心**：与嘉宾对话

**适合**：
- 行业访谈
- 经验分享
- 主题讨论

### 3. 共同主持

**核心**：多人对话

**适合**：
- 话题讨论
- 观点碰撞
- 轻松闲聊

## 从博客到播客

### 谈话要点模板

```markdown
# 播客标题：[主题] - [吸引点]

## 开场（1-2分钟）
- 引入：为什么聊这个话题
- 背景：这篇博客的来源
- 预告：听众将收获什么

## 核心内容

### 要点 1：[标题]
- 博客中的核心观点
- 延伸：额外例子或经历
- 反思：现在是否有不同看法

### 要点 2：[标题]
- 博客中的核心观点
- 延伸：额外例子或经历
- 争议：不同观点

### 要点 3：[标题]
- 博客中的核心观点
- 延伸：额外例子或经历
- 实践：如何应用

## 延伸讨论

### 个人经历
- 与主题相关的个人故事
- 失败/成功的经验

### 反面思考
- 这个观点的局限性
- 不同立场的人会怎么说
- 哪些情况不适用

## 总结（1-2分钟）
- 核心要点回顾
- 给听众的行动建议
- 鼓励实践/思考

## 结尾
- 下期预告
- 听众互动方式
- 订阅/关注引导
```

## 与博客的区别

| | 博客 | 播客 |
|---|------|------|
| 语言 | 书面语 | 口语化 |
| 结构 | 严谨 | 更自由 |
| 细节 | 完整 | 可跳跃 |
| 情感 | 克制 | 更直接 |
| 延伸 | 精炼 | 可发散 |

### 从博客到播客的转化

```
博客内容
    │
    → 口语化改写
    → 添加个人经历
    → 加入延伸讨论
    → 允许更自由的表达
    → 录制播客
```

**可以添加的内容**：
- 博客没展开的故事
- 写作时的思考过程
- 现在的不同看法
- 与读者的互动经历
- 相关的趣闻轶事

## 录制要点

### ✅ 应该做的

| 要点 | 说明 |
|------|------|
| 自然对话 | 像和朋友聊天 |
| 适当停顿 | 让听众消化 |
| 个人经历 | 增加真实感 |
| 具体例子 | 比抽象观点更好 |
| 开放讨论 | 承认不确定性 |

### ❌ 避免的

| 避免 | 原因 |
|------|------|
| 朗读博客 | 失去播客的魅力 |
| 过度准备 | 显得不自然 |
| 太正式 | 失去亲密感 |
| 没有观点 | 内容空洞 |
| 太长无内容 | 听众流失 |

## 发布建议

### 时长建议

| 内容类型 | 建议时长 |
|---------|---------|
| 短分享 | 15-20 分钟 |
| 深度讨论 | 30-45 分钟 |
| 访谈 | 45-60 分钟 |

### 发布时间

- 固定发布时间，培养听众习惯
- 早间发布适合通勤收听
- 周末发布适合深度内容
