# 博客写作通用规范

## 概述

本文件定义所有博客类型共通的基础规则。具体类型（技术、艺术、个人随笔等）有各自的补充规范，请根据内容类型选择：

```
博客类型路由
├── 技术类 → domain-technical
├── 艺术/摄影/音乐/电影 → domain-artistic
├── 设计 → domain-design
├── 文学/读书/写作 → domain-literary
├── 个人随笔/生活分享/吐槽 → domain-personal
└── 周刊/资讯 → voice-weekly
```

---

## 核心原则

### 1. 写作的本质

写作不是「产出内容」，而是**表达真实**。

- 你有话想说，才写
- 你说的是你想说的，不是「读者想听的」
- 你不确定的，可以说不确定

### 2. 类型决定规则

不同类型的博客有不同的规则优先级：

| 原则 | 技术博客 | 艺术博客 | 个人随笔 |
|------|---------|---------|---------|
| 准确性 | 必须确保 | 主观即可 | 真实即可 |
| 清晰度 | 重要 | 视情况 | 可以模糊 |
| 情感表达 | 少量 | 核心 | 核心 |
| 结构 | 有逻辑 | 自由 | 自由 |

**选择类型后，请阅读对应的子类型规范。**

## 通用写作规范

以下规则适用于所有博客类型。

### 1. 语言基础

#### 中英文混排

- 英文/数字与中文之间保留一个空格
- 使用全角标点

| ❌ 错误 | ✅ 正确 |
|--------|--------|
| OpenAI发布了新模型 | OpenAI 发布了新模型 |
| 支持5种语言 | 支持 5 种语言 |

#### 标点符号

- 使用中文标点
- 省略号用「……」（六个点）或「...」（三个英文点）
- 破折号根据语境使用，不要滥用

### 2. 段落与结构

#### 段落长度

- 每段表达一个主要意思
- 长段落用空行分隔
- 适当使用小标题划分

#### 开头

开头决定读者会不会继续读下去：

| 开头类型 | 示例 |
|---------|------|
| 场景切入 | 「凌晨三点，我还在调试这个 bug。」 |
| 问题切入 | 「你有没有遇到过这种情况：...」 |
| 观点切入 | 「我不喜欢 X。不是因为它不好，而是...」 |
| 直入主题 | 「这篇文章解决的问题是：...」 |

**避免的开头**：
- 「在当今时代...」
- 「随着...的发展...」
- 「众所周知...」

### 3. 关于「去 AI 味」

「去 AI 味」的核心是**写得像人**，但不同类型有不同的执行方式：

#### 通用去 AI 味规则

| 规则 | 说明 |
|------|------|
| 删除填充短语 | 「值得注意的是」「此外」「首先」等 |
| 避免公式结构 | 不要每段都用相同的句式 |
| 变化节奏 | 长短句混合，不要整齐划一 |
| 具体代替抽象 | 说「用了 3 小时」不说「花了很长时间」 |

#### 类型差异

| 规则 | 技术类 | 艺术类 | 个人随笔 |
|------|--------|--------|---------|
| 用事实替代形容词 | ✅ 适用 | ⚠️ 部分适用 | ❌ 不适用 |
| 删除「金句」 | ✅ 适用 | ❌ 不适用 | ⚠️ 部分适用 |
| 避免比喻 | ✅ 适用 | ❌ 不适用 | ❌ 不适用 |

**详细规则请参考各子类型文件。**

## 常见问题

### 关于语气

**问：博客应该用什么语气？**

答：取决于类型和你的个人风格。没有「正确」的语气。

| 类型 | 常见语气 |
|------|---------|
| 技术博客 | 专业但平易 |
| 艺术博客 | 感性、有审美意识 |
| 个人随笔 | 随意、真诚 |

### 关于「我」

**问：可以用「我」吗？**

答：可以，但方式不同。

| 类型 | 「我」的使用 |
|------|-------------|
| 技术博客 | 少用，「我遇到的问题是...」用于引入问题 |
| 艺术博客 | 可以用，「我看到...」「我感到...」 |
| 个人随笔 | 核心词汇，但不要自我表演 |

### 关于结论

**问：必须有结论吗？**

答：不一定。

| 类型 | 是否需要结论 |
|------|-------------|
| 教程 | 需要，总结步骤 |
| 分析文章 | 需要，但可以是开放的 |
| 个人随笔 | 不需要，可以留白 |
| 艺术赏析 | 不需要，保持开放性 |

## 插图与可视化规范

### 插图类型

#### 1. 实际图片

适用于：
- 摄影、艺术作品展示
- 实际界面截图
- 实物照片
- 实验结果

#### 2. 示意图

适用于：
- 流程说明
- 概念解释
- 结构展示
- 对比说明

#### 3. Mermaid 图表

适用于：
- 流程图
- 架构图
- 时序图
- 甘特图
- 状态图

### 图片描述预留规范

## 路由指南

确定博客类型后，阅读对应的详细规范：

### 按类型路由

| 类型 | 技能名称 |
|------|------|
| 教程、技术分析、实践笔记 | `domain-tech` |
| 摄影、音乐、电影、艺术赏析 | `domain-artistic` |
| UI/UX、产品设计、视觉设计 | `domain-design` |
| 读书笔记、写作思考、随笔散文 | `domain-literary` |
| 个人随笔、生活分享、吐槽 | `domain-personal` |

### 分发路由

博客完成后，如需分发到社交媒体平台，请使用对应的 `platform-` 平台技能（如 `platform-social`、`platform-video`、`platform-audio-podcast` 等），并叠加 `voice-social` 技能进行改写。
