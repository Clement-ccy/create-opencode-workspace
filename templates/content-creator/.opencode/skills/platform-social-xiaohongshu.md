# 小红书内容规范

## 平台特性

- 视觉驱动，文案需配合图片/视频。
- 情绪价值 > 信息密度。
- **语调**：亲切、种草风、有个人特色
- **形式**：图文结合，视觉优先
- **特点**：重互动、重收藏、话题标签多

---

## 内容结构

### 标准结构

```
[标题 - 带情绪/数字/痛点]

[封面图 - 3:4 竖版，有文字信息]

[正文 - 300-800字]
• 要点 1
• 要点 2
• 要点 3

[互动邀请]
[话题标签]
```

### 标题公式

| 类型 | 公式 | 示例 |
|------|------|------|
| 数字型 | [数字] 个 [结果] 方法 | 5 个提升效率的方法 |
| 对比型 | [之前] vs [之后] | 学会这个之前 vs 之后 |
| 痛点型 | 还在 [错误做法]？ | 还在手动整理？ |
| 种草型 | [时间] 用了 [结果] | 用了一周，真的香 |

## 内容类型

### 1. 种草/测评

分享产品、工具、资源的体验。

**结构**：
- 使用背景/场景
- 核心体验
- 优缺点
- 适合谁/不适合谁
- 购买/获取方式

**示例**：
> 用了一个月的 [工具名]，说说真实感受。
>
> 背景说一下：我是 [场景]，需要 [需求]。
>
> 先说优点：
> • [优点1]
> • [优点2]
>
> 再说缺点：
> • [缺点1]
>
> 适合：[什么样的人]
> 不适合：[什么样的人]
>
> 总体评价：[结论]

### 2. 教程/干货

分享方法、技巧、流程。

**结构**：
- 问题引入
- 步骤分解
- 常见问题
- 总结

**示例**：
> 整理了一下 [主题] 的方法，试了 [时间]，亲测有效。
>
> 具体步骤：
>
> 1️⃣ [步骤1]
> [说明]
>
> 2️⃣ [步骤2]
> [说明]
>
> 3️⃣ [步骤3]
> [说明]
>
> 常见问题：
> Q: [问题]
> A: [回答]
>
> 希望对你有帮助~

### 3. 日常/分享

分享生活、工作日常。

**结构**：
- 场景/事件
- 感受/想法
- 互动

**示例**：
> 今天发生了一件小事。
>
> [描述事件]
>
> [感受]
>
> 你们有没有类似的经历？

## 写作要点

### ✅ 应该做的

| 要点 | 说明 |
|------|------|
| 标题有吸引力 | 决定点击率 |
| 配图要有信息 | 封面是第一印象 |
| 分点列述 | 便于快速浏览 |
| 真实体验 | 不硬广、不假种草 |
| 引导互动 | 评论/收藏/点赞 |

### ❌ 避免的

| 避免 | 原因 |
|------|------|
| 纯广告 | 用户反感 |
| 夸大效果 | 失去信任 |
| 没有个人观点 | 显得假 |
| 太长不看 | 手机阅读为主 |

---

## 话题标签

- 数量：5-15 个
- 位置：正文末尾
- 混合：热门 + 长尾 + 品牌标签

**示例**：
```
#效率工具 #notion #时间管理 #职场必备 #干货分享 #学习方法
```

---

## 配图规范

| 元素 | 要求 |
|------|------|
| 尺寸 | 3:4 竖版（1080×1440px） |
| 文字 | 必须有，突出核心信息 |
| 风格 | 统一、有辨识度 |
| 数量 | 1-9 张，首图最重要 |

---

## 发布时间

| 时间段 | 推荐度 |
|--------|--------|
| 晚 7-10 点 | ✅ 最佳 |
| 周末全天 | ✅ 良好 |
| 午休时间 | ✅ 良好 |

---

## 与博客的关系

```
博客 → 小红书
     → 提取 3-5 个要点
     → 重新组织成小红书风格
     → 制作封面图
     → 添加话题标签
```

**注意**：小红书用户习惯与博客不同，需要：
- 更短的篇幅
- 更直观的视觉
- 更多的互动引导
