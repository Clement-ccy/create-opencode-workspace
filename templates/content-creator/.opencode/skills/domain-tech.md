# 技术类博客

## 类型定位

技术博客的核心是**解决问题**和**分享知识**。读者来到这里，是想学到东西的。

---

## 核心特质

- **准确**：代码能跑，结论经得起验证
- **清晰**：步骤明确，逻辑连贯
- **实用**：解决实际问题，不是炫技
- **诚实**：不懂的说不懂，局限的要说明

---

## 子类型

### 1. 教程 (Tutorial)

教会读者一项具体技能。

**结构**：
```markdown
# [达成目标]：[时间/难度]

## 前置条件
- 需要 X 基础
- 需要准备 Y 环境

## 开始之前
[简短的背景/为什么学这个]

## 步骤 1：[动作]
[代码 + 解释]

## 步骤 2：[动作]
[代码 + 解释]

## 常见问题
Q: [问题]
A: [解决方案]

## 总结
[回顾要点，推荐下一步]
```

**写作要点**：

| 要点 | 说明 |
|------|------|
| 代码可运行 | 不要假设读者能补全 |
| 解释「为什么」 | 不只是「怎么做」，还有「为什么这么做」 |
| 预告结果 | 每步之后，告诉读者应该看到什么 |
| 标注版本 | 环境/版本差异是教程最大的敌人 |

### 2. 深度分析 (Deep Dive)

深入探讨某个技术点，面向有基础的读者。

**结构**：
```markdown
# [主题]深度解析

## 背景
[为什么这个话题重要]

## 基础概念
[快速回顾，面向有基础读者]

## 核心原理
[层层递进的解释]

## 实践验证
[代码/实验证明]

## 与其他方案的对比
[优劣分析]

## 总结
[核心要点]
```

**写作要点**：

- 不要从零解释，假设读者有基础
- 用代码/实验证明观点，不只是理论
- 可以有个人判断，但要说明理由

### 3. 实践笔记 (Practice Note)

记录实际项目中遇到的问题和解决方案。

**结构**：
```markdown
# [问题/场景]：从[方案A]到[方案B]

## 场景
[具体项目背景]

## 问题
[遇到的具体问题]

## 尝试的方案
### 方案 1：[名称]
[为什么选它 + 失败原因]

### 方案 2：[名称]
[为什么选它 + 失败原因]

### 最终方案：[名称]
[为什么成功]

## 代码/实现
[关键代码片段]

## 经验总结
[可复用的经验]
```

**写作要点**：

- 记录失败很重要，读者可能遇到同样问题
- 场景要具体，不要抽象化
- 经验要可迁移

### 4. 工具/项目介绍

介绍一个工具、库或项目。

**结构**：
```markdown
# [项目名]：[一句话价值]

## 是什么
[简短定义]

## 解决什么问题
[痛点描述]

## 如何使用
[快速上手代码]

## 核心特性
[3-5 个亮点]

## 适用场景
[什么情况下用]

## 局限性
[什么情况下不用]
```

**写作要点**：

- 开头就要说清楚「这东西干嘛的」
- 代码要能直接跑
- 承认局限，不要只说优点

## 技术写作的特殊规范

### 代码规范

- **可运行**：代码应该是完整可运行的，或明确标注省略部分
- **有注释**：关键行添加注释
- **版本标注**：注明语言/框架版本

````markdown
```python
# Python 3.10+
def solve(data: list[int]) -> int:
    # 使用生成器表达式提高内存效率
    return sum(x * 2 for x in data if x > 0)
```
````

### 术语使用

| 情况 | 处理方式 |
|------|---------|
| 首次出现专业术语 | 简短解释或给链接 |
| 通用术语 | 直接使用 |
| 自造词/非标准用法 | 明确说明 |

### 不确定性的表达

技术领域总有不确定的东西，诚实表达：

| 表达 | 适用情况 |
|------|---------|
| 「我的理解是...」 | 个人推断 |
| 「可能的原因是...」 | 猜测原因 |
| 「不确定，建议查阅...」 | 真的不懂 |
| 「在某些情况下...」 | 有条件成立 |

---

## 与个人随笔的区别

技术博客要**减少主观情绪的表达**：

| | 技术博客 | 个人随笔 |
|---|---------|---------|
| 情绪表达 | 少，点到为止 | 可以多 |
| 不确定性 | 要明确标注 | 可以模糊 |
| 结构 | 清晰有逻辑 | 可以跳跃 |
| 读者期待 | 学到东西 | 产生共鸣 |

---

## 检查清单

- [ ] 代码能跑吗？
- [ ] 环境要求说清楚了吗？
- [ ] 每个步骤有「为什么」的解释吗？
- [ ] 局限性说明了吗？
- [ ] 版本信息标注了吗？
- [ ] 有没有跳过关键步骤？
