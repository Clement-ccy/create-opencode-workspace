# Twitter/X 内容规范

## 平台特点

- **受众**：多样化，信息快速流动
- **语调**：简洁、有力、可以直接
- **长度**：单条 100-250 字符，推文串 5-15 条
- **特点**：实时性、话题性、可讨论

---

## 内容形式

### 1. 单条推文

**核心**：一个观点，简洁表达。

```
[核心观点] [可选：链接/图片] #话题标签
```

**示例**：
> 写了三年技术博客，最大的收获不是涨粉。
>
> 是学会了把复杂的事情说清楚。
>
> 这个技能比我想象的更有用。

### 2. 推文串 (Thread)

**核心**：展开一个观点，有结构。

**结构模板**：
```
1/ [开篇 Hook - 为什么这重要]

2/ [背景/问题]

3/ [核心观点 1]

4/ [核心观点 2]

5/ [核心观点 3]

6/ [案例/证据]

7/ [反面思考]

8/ [总结]

N/ [收尾 + 可选链接]
```

**示例**：
> 1/ 昨天重构了一个「看起来没问题」的代码。
>
> 结果发现问题比想象的多。
>
> 几个反思：
>
> 2/ 第一个问题是命名。
>
> 变量名本身没问题，但放在一起就乱了。
>
> user_data, userData, userDat... 你能分清吗？
>
> 3/ 第二个问题是注释。
>
> 注释写了「这里需要优化」，但没写为什么。
>
> 三个月后，没人知道要优化什么。
>
> 4/ 第三个问题是测试。
>
> 测试覆盖率很高，但都是 happy path。
>
> 边界情况？没有。
>
> 5/ 重构花了两天。
>
> 如果一开始就写清楚，可能只要两小时。
>
> 6/ 三个教训：
> • 命名要一致，不只是「有意义」
> • 注释写「为什么」，不只是「是什么」
> • 测试想「什么会出错」，不只是「正常情况」

## 写作要点

### ✅ 应该做的

| 要点 | 说明 |
|------|------|
| 第一条就抓住注意 | 决定用户会不会展开 |
| 每条一个意思 | 不要塞太多 |
| 可以有个性 | 幽默/讽刺/直接都可以 |
| 用编号标记推文串 | 「1/」「2/」帮助阅读 |

### ❌ 避免的

| 避免 | 原因 |
|------|------|
| 无意义转发 | 纯刷屏 |
| 过度话题标签 | 1-2 个足够 |
| 每条都是链接 | 没人点 |
| 太长没换行 | 手机上难读 |

---

## 话题标签

- 单条推文：1-2 个标签，放末尾
- 推文串：可以在第一条放标签

**常用标签**：
- 中文：#技术 #设计 #产品 #写作
- 英文：#Tech #AI #Design

---

## 发布时间

| 时间段 | 推荐度 |
|--------|--------|
| 工作日 9-10 点 | ✅ 最佳 |
| 工作日晚 7-9 点 | ✅ 良好 |
| 周末 | ⚠️ 视内容而定 |

---

## 与博客的关系

Twitter/X 内容可以是：

```
博客 → Twitter/X
     → 提取 3-5 个核心点
     → 改写成推文串
     → 最后一条附博客链接
```

或：

```
Twitter/X → 博客
         → 推文串反响好
         → 扩展成完整博客
```

## 检查清单

### 单条推文

- [ ] 核心观点清楚吗？
- [ ] 有没有可以删掉的词？
- [ ] 标签数量适中吗？

### 推文串

- [ ] 第一条够吸引人吗？
- [ ] 每条一个主要意思？
- [ ] 有没有编号？
- [ ] 最后一条有收尾吗？
