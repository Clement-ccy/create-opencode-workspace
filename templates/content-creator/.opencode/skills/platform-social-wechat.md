# 微信公众号内容规范

## 平台特点

- **受众**：私域流量，订阅者为主
- **语调**：可以更个人化，与读者建立长期关系
- **形式**：长文为主，图文混排
- **特点**：阅读深度高，转发传播

---

## 内容结构

### 标准结构

```
[标题 - 30字以内]

[封面图 - 900×500px 或 900×383px]

[摘要 - 50字内，显示在推送列表]

---

[正文]

## [小标题]

[段落]

## [小标题]

[段落]

---

[结尾/互动]

关注我，一起 [价值主张]
```

---

## 标题策略

### 公式

| 类型 | 公式 | 示例 |
|------|------|------|
| 问题型 | [问题]？ | 为什么你总是完不成计划？ |
| 数字型 | [数字] 个 [结果] | 5 个提升写作效率的方法 |
| 故事型 | [故事开头]... | 那个天天加班的人，后来怎样了 |
| 观点型 | [观点] | 技术人不需要学管理，真的吗？ |

### 注意

- 不做标题党：标题要能兑现
- 30 字以内：完整显示
- 可以有悬念：但不要误导

## 正文写作

### 段落规范

| 要素 | 规范 |
|------|------|
| 段落长度 | 3-5 行，不超过一屏 |
| 小标题 | 每 300-500 字一个小标题 |
| 配图 | 每 500-800 字一张图 |
| 强调 | 关键信息加粗 |

### 排版风格

- 统一字体大小（建议正文 15-16px）
- 行间距 1.5-1.75
- 段间距明显
- 主色调不超过 3 种

### 开头技巧

| 开头类型 | 示例 |
|---------|------|
| 场景引入 | 「上周，一个读者问我...」 |
| 问题引入 | 「你有没有遇到过这种情况：...」 |
| 观点引入 | 「我觉得 X 是被高估的」 |
| 直入主题 | 「这篇文章解决的问题是：...」 |

**避免的开头**：
- 「大家好，我是...」（除非是新号介绍）
- 「在当今时代...」
- 「众所周知...」

### 结尾方式

| 结尾类型 | 适用情况 |
|---------|---------|
| 总结回顾 | 教程、干货类 |
| 开放讨论 | 观点、思考类 |
| 行动呼吁 | 希望读者做某事 |
| 留白 | 个人随笔、艺术类 |

## 配图规范

### 封面图

| 尺寸 | 用途 |
|------|------|
| 900×500px（16:9） | 首条推送 |
| 900×383px（2.35:1） | 非首条推送 |

### 正文配图

- 与内容相关
- 风格统一
- 不要水印（或统一水印）
- 可以是：截图、示意图、照片、插图

## 互动设计

### 引导关注

```
我是 [名字]，[一句话介绍]
分享 [内容类型]
关注我，一起 [价值]
```

### 引导互动

- 问题讨论：「你怎么看？评论区见」
- 转发引导：「觉得有用，转给需要的人」
- 点赞鼓励：「点赞让我知道你在看」

## 与博客的关系

微信公众号可以是：

### 博客的同步发布

```
博客内容
    │
    → 添加封面图
    → 优化排版（加粗、分段）
    → 添加引导语
    → 发布
```

### 博客的深度版

```
博客内容
    │
    → 扩展案例
    → 增加配图
    → 添加读者互动
    → 发布
```

### 博客的精选版

```
多篇博客
    │
    → 提取精华
    → 整合成专题
    → 发布
```
