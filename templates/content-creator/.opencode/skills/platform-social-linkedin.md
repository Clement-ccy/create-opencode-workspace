# LinkedIn 内容规范

## 平台特点

- **受众**：职场人士、行业同行、潜在雇主/合作伙伴
- **语调**：专业但有温度，不过于正式
- **长度**：1300-2000 字符为佳
- **格式**：多换行，便于移动端阅读

---

## 内容结构模板

### 标准结构

```
🎯 [Hook/开头第一句 - 抓住注意力]

[背景/场景 - 1-2句话说明情况]

核心观点：
• 要点 1（具体细节）
• 要点 2（可量化的结果）
• 要点 3（意外发现）

[简短故事或案例说明]

[总结/启发]

[问题引发讨论]？

#话题标签1 #话题标签2 #话题标签3
```

### 示例

> 昨天面试了一个候选人，他的回答让我停下了笔。
>
> 我问他：「你最大的失败是什么？」
>
> 他没有说「我太追求完美」这种标准答案。他说：「我低估了人的复杂性。」
>
> 然后他讲了一个项目失败的故事——不是技术失败，是沟通失败。
>
> 三点反思：
> • 透明不等于有效沟通
> • 共识需要时间，不能假设
> • 好的意图不保证好的结果
>
> 最后他说：「我现在会多问一句：你理解的是...吗？」
>
> 有时候，最简单的改变最有效。
>
> 你在团队沟通中遇到过类似的「我以为」时刻吗？

## 内容类型

### 1. 经验分享

分享工作中的经验、教训或洞察。

**结构**：
- 场景引入
- 问题/挑战
- 你的处理方式
- 反思/收获
- 开放性问题

### 2. 行业观察

对行业趋势、变化的观察和思考。

**结构**：
- 观察到什么
- 为什么重要
- 可能的影响
- 你的判断（可以不确定）

### 3. 个人故事

职场相关的个人经历。

**结构**：
- 故事背景
- 关键事件
- 转折/变化
- 你的成长

### 4. 观点表达

对某个话题的看法。

**注意**：
- 可以有立场，但要承认复杂性
- 避免过于绝对的判断
- 欢迎不同意见

## 写作要点

### ✅ 应该做的

| 要点 | 说明 |
|------|------|
| 第一句抓住注意 | 决定用户会不会点「展开」 |
| 使用换行 | 移动端体验优先 |
| 具体而非抽象 | 说「花了 3 个月」不说「花了很长时间」 |
| 承认不确定 | 「我不确定」「也许」是诚实的表现 |
| 引发讨论 | 结尾可以提问 |

### ❌ 避免的

| 避免 | 原因 |
|------|------|
| 过于正式 | LinkedIn 不等于简历 |
| 纯粹的自我宣传 | 没人想看广告 |
| 空洞的激励 | 「相信自己」没有信息量 |
| 过长的段落 | 移动端阅读体验差 |

## 话题标签

- 使用 3-5 个标签
- 混合热门标签和精准标签
- 标签放在文末

**常用标签示例**：
- 行业类：#Tech #AI #Design #Marketing
- 主题类：#Leadership #Career #Productivity
- 讨论类：#Thoughts #Perspective

## 发布时间

| 时间段 | 推荐度 |
|--------|--------|
| 工作日上午 8-10 点 | ✅ 最佳 |
| 工作日中午 12-1 点 | ✅ 良好 |
| 周二至周四 | ✅ 推荐 |
| 周末 | ⚠️ 效果较差 |

## 与博客的关系

LinkedIn 内容可以是：

- 博客的精简版：核心观点 + 讨论邀请
- 博客的延伸：博客没展开的角度
- 独立内容：不适合发博客的职场观察

**分发策略**：
```
博客 → LinkedIn
     → 提取核心观点
     → 精简到 1500 字符以内
     → 添加讨论性问题
     → 可附链接
```

## 检查清单

- [ ] 第一句是否足够吸引人？
- [ ] 段落是否足够短（移动端友好）？
- [ ] 有没有具体的故事或数据？
- [ ] 结尾是否引发讨论？
- [ ] 话题标签是否合适？
