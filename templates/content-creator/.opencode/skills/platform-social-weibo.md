# 微博内容规范

## 平台特点

- **受众**：广泛，公共舆论场
- **语调**：简洁有力，可以直白
- **形式**：短内容为主，配图加分
- **特点**：传播快、话题性、互动多

---

## 内容形式

### 1. 短微博

**核心**：140 字内表达清楚。

```
[核心内容] [图片/视频] #话题#
```

**示例**：
> 写代码的时候，命名是最难的事之一。
>
> 不是起不出来，是不确定这个变量三年后的人能不能看懂。
>
> #程序员日常

### 2. 长微博

**核心**：超出 140 字，可以展开说。

```
[开头抓眼]

[主体内容]

[总结/互动]

#话题#
```

**示例**：
> 面试的时候，我问候选人：你最大的失败是什么？
>
> 他说：我低估了沟通的复杂性。
>
> 然后他讲了一个项目失败的故事。不是因为技术不行，是因为「我以为对方懂了」。
>
> 这个回答打动我。不是因为他有失败，而是因为他知道为什么失败。
>
> 有时候，最简单的反思最有力。
>
> #职场 #面试

### 3. 九宫格

**核心**：图片为主，文字引导。

```
[一句话概括]

[九张图展示内容]

#话题#
```

---

## 写作要点

### ✅ 应该做的

| 要点 | 说明 |
|------|------|
| 开头抓眼 | 第一句决定是否继续看 |
| 配图加分 | 图片/视频提升 2-3 倍互动 |
| 话题标签 | 帮助被发现，1-2 个 |
| 可以直接 | 微博允许直白表达 |

### ❌ 避免的

| 避免 | 原因 |
|------|------|
| 纯转发无评论 | 没有增量价值 |
| 过多话题标签 | 1-2 个足够 |
| 只发链接 | 没人点 |
| 过度营销 | 用户反感 |

---

## 话题标签

- 数量：1-2 个
- 位置：微博末尾
- 选择：热门话题 + 垂直话题

**常用话题示例**：
- #程序员 #技术 #设计
- #职场 #工作 #生活

---

## 配图规范

| 形式 | 说明 |
|------|------|
| 单图 | 重点突出 |
| 四图 | 对比或步骤 |
| 九图 | 系列、过程、详情 |
| 视频 | 最高传播形式 |

**图片尺寸**：
- 推荐 16:9 或 1:1
- 不要过于复杂的文字

---

## 发布时间

| 时间段 | 推荐度 |
|--------|--------|
| 午 12-13 点 | ✅ 午休高峰 |
| 晚 18-22 点 | ✅ 最佳时段 |
| 早 8-9 点 | ✅ 通勤时间 |

---

## 与博客的关系

```
博客 → 微博
     → 提取核心观点
     → 浓缩到 140 字或 300 字
     → 配图或链接
     → 添加话题标签
```

**策略**：
- 技术博客：提取一个要点 + 配图
- 观点博客：核心观点 + 讨论邀请
- 个人随笔：可以全文同步

---

## 检查清单

- [ ] 第一句够吸引人吗？
- [ ] 核心内容清楚吗？
- [ ] 有没有配图？
- [ ] 话题标签合适吗？
- [ ] 长度合适吗？
