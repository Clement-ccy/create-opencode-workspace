# 社交媒体平台分发规范

此技能用于将长文内容转化为各社交媒体平台的最佳发布格式。

---

## 适用平台

| 平台 | 类型 | 核心特征 |
|------|------|---------|
| Twitter/X | 短文/Thread | 实时、观点导向、传播快 |
| 微信公众号 | 深度长文 | 私域流量、深度阅读 |
| 知乎 | 问答/长文 | 结构化、专业、长篇幅 |
| CSDN | 技术博客 | 代码示例、教程类 |
| 小红书 | 卡片图文 | 视觉导向、种草/干货 |

---

## 平台适配矩阵

| 维度 | Twitter/X | 微信公众号 | 知乎 | CSDN | 小红书 |
|------|-----------|-----------|------|------|--------|
| 标题风格 | 直接/悬念 | 吸引点击 | 疑问/立场 | 技术关键词前置 | Emoji + 痛点 |
| 字数 | 280字/条，可 Thread | 1500-3000字 | 2000-5000字 | 1000-4000字 | 300-800字 |
| 配图 | 可选 | 3-6张 | 可选截图 | 代码截图 | 封面 + 内页图 (3:4) |
| 语调 | 轻快/锐利 | 深度/个人化 | 专业/逻辑 | 专业/教程 | 感性/亲切/闺蜜感 |
| 互动引导 | 提问/投票 | 关注/评论 | 点赞/收藏 | 收藏/关注 | 收藏/关注/评论 |
| 标签 | 1-3个 # | 无 | 3-5个话题 | 3-5个技术标签 | 5-10个 # |

## 平台特定输出规范

### Twitter/X

```markdown
【Thread 格式】

1/ [Hook - 抛出核心观点或反常识结论]

2/ [展开论点 1 - 最多 280 字]

3/ [展开论点 2]

...

10/ [总结 + 你的观点]

11/ [CTA - 如 "觉得有用就转发吧"]

#标签1 #标签2
```

**规则**：
- 每条推文不超过 280 字符
- Thread 总条数控制在 5-12 条
- 第 1 条必须有 Hook，最后 1 条有 CTA
- 可在中间穿插图片或引用推文

### 微信公众号

```markdown
【标题】20字以内，有吸引力

【引言】前3行抓住读者（场景/问题/观点切入）

## 小标题1
正文段落，每段不超过3行...

## 小标题2
正文段落...

## 结尾
总结 + 互动引导（提问/投票）

---
关注公众号获取更多内容
```

**规则**：
- 适合移动端阅读，每段不超过 3 行
- 使用引用块突出关键信息
- 代码需转为图片
- 结尾必须有互动引导

### 知乎

```markdown
【标题】疑问句或明确立场句

## 1. 核心观点

详细分析...

## 2. 论据支撑

数据、案例、引用...

## 3. 补充说明

其他角度的思考...

---

**相关阅读**：[链接]
```

**规则**：
- 使用编号小标题，便于跳读
- 允许长篇幅但需有目录感
- 引用数据需注明来源
- 可附个人简介/专栏链接

### CSDN

```markdown
【标题】技术关键词前置，如「Python 实现 XXX 的 5 种方法」

## 1. 前言/背景

为什么需要这个方案...

## 2. 实现步骤

### 2.1 环境准备

​```bash
pip install xxx
​```

### 2.2 核心代码

​```python
# 代码示例
​```

### 2.3 结果展示

截图/输出结果...

## 3. 总结

简要总结 + 下一步
```

**规则**：
- 代码块必须完整，语言标注明确
- 每个步骤配截图或输出结果
- 添加 3-5 个技术标签
- URL slug 使用英文关键词

### 小红书

```markdown
【标题】[Emoji] 痛点/价值承诺，20字以内

📝 前言（直接抛出痛点）

✅ 要点 1
一句话说明

✅ 要点 2
一句话说明

✅ 要点 3
一句话说明

💡 总结
一句话总结

📢 互动引导（提问/求建议）

#标签1 #标签2 #标签3 #标签4 #标签5 #标签6
```

**规则**：
- 大量使用 Emoji 做视觉引导
- 每个要点一行，不要大段文字
- 配图必须是 3:4 竖图，首图要有冲击力
- 标签至少 5 个，多则 10 个
- 结尾有明确的收藏/关注引导

---

## 转化原则

1. **不要跨平台复制粘贴**：每个平台的输出必须是独立适配的
2. **降维表达**：专业术语转化为日常比喻
3. **信息压缩**：从长文中提取 1-3 个最核心的洞察
4. **CTA 差异化**：每个平台的互动引导方式不同

## 输出格式

```
Assets/[Category]/[name]/distributions/
├── twitter/[name]_twitter.md
├── wechat/[name]_wechat.md
├── zhihu/[name]_zhihu.md
├── csdn/[name]_csdn.md
└── xiaohongshu/[name]_xiaohongshu.md
```

每个文件必须包含对应的 Notion frontmatter。

## 分发检查清单

- [ ] 是否为每个平台生成了独立的适配内容？
- [ ] Twitter 是否控制在 280 字/条且有清晰的 Thread 结构？
- [ ] 微信公众号段落是否适合移动端？
- [ ] 知乎内容是否有清晰的目录结构？
- [ ] CSDN 是否包含完整的代码块？
- [ ] 小红书配图是否是 3:4 竖图？
- [ ] 每个平台的 CTA 是否合适？
- [ ] 标签/话题是否符合各平台习惯？
