# 文字平台分发排版规范

此技能用于将内容适配到文字为主的发布平台，解决跨语言、跨平台的排版和阅读体验问题。

---

## 适用平台

| 平台 | 核心特征 | 内容偏好 |
|------|---------|---------|
| 微信公众号 | 移动端阅读为主、私域流量 | 深度长文、个人化叙述 |
| 知乎 | 问答驱动、结构化 | 专业分析、长篇回答 |
| CSDN | 技术博客、代码展示 | 教程、技术笔记、代码示例 |

---

## 跨语言排版规范

### 中文发布

| 规则 | 说明 |
|------|------|
| 全角标点 | 使用中文标点符号（，。！？：；） |
| 中英文空格 | 英文/数字与中文之间加空格 |
| 章节标题 | 使用 `#` 或 `##`，公众号建议手动加粗代替 Markdown 标题 |
| 代码块 | 使用缩进或引用块，公众号需转为图片 |
| 链接 | 知乎/CSDN 可用 Markdown 链接；公众号需用「阅读原文」或截图 |

### 英文发布

| 规则 | 说明 |
|------|------|
| 半角标点 | 使用英文标点（,.!?） |
| 段落 | 每段不超过 3-4 句 |
| 标题 | Title Case 或 Sentence Case |
| 链接 | 内嵌到关键词，不单独列出 |

### 中英双语版本

- **策略**：先写主语言版本，再做 Transcreation（文化适配重写），**禁止**直译
- **输出**：两个独立文件，分别存放在 `distributions/` 的对应语言目录下
- **文件命名**：`[name]_zh.md` 和 `[name]_en.md`

## 平台特定适配

### 微信公众号

| 规则 | 说明 |
|------|------|
| 标题 | 20 字以内，有吸引力，含关键词 |
| 开头 | 前 3 行必须抓住注意力（场景/问题/观点切入） |
| 段落 | 每段不超过 3 行（移动端换行频繁） |
| 图片 | 3:4 竖图优先，宽图宽度不超过屏幕 |
| 引用 | 用引用块格式，突出金句或数据 |
| 结尾 | 必须有互动引导（提问/投票/关注） |
| 排版工具 | 可用 Markdown → 公众号编辑器转换工具 |
| 代码处理 | 代码需转为图片或用代码排版工具 |

### 知乎

| 规则 | 说明 |
|------|------|
| 标题 | 疑问句或明确立场句 |
| 结构 | 使用小标题 + 编号列表，便于跳读 |
| 深度 | 允许长篇幅（2000-5000 字），但需有目录感 |
| 引用 | 引用数据或来源时附链接 |
| 签名 | 结尾可附个人简介/专栏链接 |

### CSDN

| 规则 | 说明 |
|------|------|
| 标题 | 技术关键词前置，如「Python 实现 XXX 的 5 种方法」 |
| 结构 | 教程型：步骤编号 + 代码块 + 结果截图 |
| 代码 | 必须有代码块，语言标注明确 |
| 标签 | 添加 3-5 个技术标签 |
| 排版 | 使用 CSDN 的 Markdown 编辑器 |
| SEO | URL slug 使用英文关键词 |

## 输出格式

内容适配后保存到：

```
Assets/[Category]/[name]/distributions/
├── wechat/[name]_wechat.md
├── zhihu/[name]_zhihu.md
└── csdn/[name]_csdn.md
```

每个文件必须包含对应的 Notion frontmatter（参考 `frontmatter-notion-property` skill）。

## 排版检查清单

- [ ] 标题是否适合该平台的标题风格？
- [ ] 段落长度是否适配移动端/PC 端？
- [ ] 中英文混排格式是否正确？
- [ ] 代码块是否完整且语法高亮标注？
- [ ] 图片是否按平台要求处理（比例、大小）？
- [ ] 是否有平台特定的互动引导（CTA）？
- [ ] 链接是否正确嵌入？
