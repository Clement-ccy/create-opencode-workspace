# Voice Optimization — 全局基础规范与路由系统

本文件是所有内容创作技能的**根路由**。当进行内容创作时，必须先确定以下三件事：

1. **内容类型** (Type) → 决定基本格式和写作规范
2. **领域上下文** (Domain) → 决定语调、专业度、表达方式
3. **发布平台** (Platform) → 决定排版、长度、分发格式

---

## 路由系统

### Step 1: 确定内容类型

| 内容类型 | 路由目标 | 说明 |
|---------|---------|------|
| 博客/长文 | `type-blog` | 800+ 字，深度文章 |
| 周刊/资讯 | `type-weekly` | 信息聚合、条目式 |
| Mlog (音乐日志) | `type-blog` + `domain-artistic` | 编曲记录、翻唱记录、音乐制作 |
| Plog (摄影日志) | `type-blog` + `domain-artistic` | 摄影作品、拍摄经历、器材体验 |
| 社交媒体分发 | `type-social` + `platform-social` | 短图文、Thread |
| 视频脚本 | `type-social` + `platform-video` | 分镜脚本、HTML PPT |

### Step 2: 确定领域上下文

| 领域 | 路由目标 | 适用场景 |
|------|---------|---------|
| 技术 | `domain-tech` | 编程、教程、技术分析、工具评测 |
| 艺术 | `domain-artistic` | 摄影、音乐、电影、艺术赏析 |
| 设计 | `domain-design` | UI/UX、产品设计、视觉设计 |
| 文学 | `domain-literary` | 读书笔记、写作思考、随笔散文 |
| 个人 | `domain-personal` | 个人随笔、生活分享、吐槽 |

### Step 3: 确定发布平台

| 平台 | 路由目标 | 核心要求 |
|------|---------|---------|
| Blog/个人网站 | 无额外路由 | 直接使用 type-blog 规范 |
| 微信公众号 | `platform-text` | 适配移动端阅读 |
| 知乎 | `platform-text` | 长文深度、结构化 |
| CSDN | `platform-text` | 技术博文格式 |
| Twitter/X | `platform-social` | Thread 格式、字数限制 |
| 小红书 | `platform-social` | 卡片排版、Emoji 密集 |
| 视频平台 (B站/YouTube) | `platform-video` | 分镜脚本、JSON recipe |

## 全局写作铁则

### 1. 去 AI 味

| 规则 | 说明 |
|------|------|
| 删除填充短语 | 去除「值得注意的是」「首先」「此外」「总之」 |
| 打破公式结构 | 不要每段都是"X 是 Y"的句式 |
| 变化节奏 | 长短句混合，不要整齐划一 |
| 具体代替抽象 | 说「用了 3 小时」不说「花了很长时间」 |
| 信任读者 | 直接陈述，不要过度解释 |
| 删除金句 | 如果听起来像可以裱起来的话，重写它 |

### 2. 中英文混排

- 英文/数字与中文之间保留一个空格
- 使用全角标点（中文语境下）
- 专有名词首次出现附中文注释

| ❌ 错误 | ✅ 正确 |
|--------|--------|
| OpenAI发布了新模型 | OpenAI 发布了新模型 |
| 支持5种语言 | 支持 5 种语言 |

### 3. 列表与序号

- 有序列表：只用 `1. `、`2. `（阿拉伯数字 + 英文句点 + 空格）
- 无序列表：只用 `- `（短横线 + 空格）
- 章节划分：用 Markdown 标题 `##` / `###`，**禁止**用 `一、`、`二、`

### 4. 人称使用

| 场景 | 推荐人称 |
|------|---------|
| 教程/工具介绍 | 第二人称「你」 |
| 个人观点/思考 | 第一人称「我」 |
| 团队公告 | 第一人称「我们」 |

### 5. 简化词池（替换 AI 高频词）

| ❌ 不要用 | ✅ 替换为 |
|----------|----------|
| 至关重要 | 重要 / 关键 |
| 深入探讨 | 讨论 / 分析 |
| 不断演变的格局 | 变化 / 趋势 |
| 充满活力 | 活跃 / 删除 |
| 宝贵的 | 有用的 / 删除 |
| 此外 | 直接删除，另起一句 |
| 值得注意的是 | 直接删除 |

## 通用检查清单

内容完成前逐项检查：

- [ ] 是否选择了正确的 type/domain/platform 路由？
- [ ] 前三句能否抓住读者？
- [ ] 有没有「值得注意的是」「此外」等 AI 味填充词？
- [ ] 中英文混排格式是否正确（空格 + 全角标点）？
- [ ] 段落是否清晰，每段一个主要意思？
- [ ] 结尾是否自然收束，而非强行总结？
- [ ] 是否有明确的互动引导（CTA）？
