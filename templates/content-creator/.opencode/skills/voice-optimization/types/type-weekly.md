# 周刊写作指南

## 核心原则

**简单 (Simple)**、**人性化 (Human)**、**清晰 (Clear)**

---

## 1. 周刊模板

```yaml
---
Name: 「[year]-Weekly[week]」[标题]
Category: Weeklys
Description: 
Tags: 
    - [Tag1]
    - [Tag2]
Area: [领域，如 Tech / AI / Design]
Role: Writer
Status: Draft
Publish Date: [YYYY-MM-DD]
Platform:
    - Article
    - Video
    - Audio
Link: "/weekly/[slug]"
Theme: teal
cover: "./[slug]-cover.jpg"
---
```

正文结构：

```markdown
# 「[year]-Weekly[week]」[标题]

视频版：[哔哩哔哩](link) ｜ [YouTube](link)

## 本期要点

- [条目2标题] [↗](link) `#2`
- [条目3标题] [↗](link) `#3`
- ...

---
```

## 3. 栏目分类

根据内容性质分组到不同栏目：

| 栏目 | 适用内容 |
|------|---------|
| **要闻** | 重大新闻、产品发布、重要合作 |
| **开发生态** | 开发工具、SDK、编程相关 |
| **行业动态** | 公司战略、开源承诺、融资 |
| **模型发布** | 新模型、版本更新、性能提升 |
| **工具推荐** | 值得一试的新工具、产品 |
| **产品应用** | 新的产品、应用新功能 |
| **技术洞察** | 新颖的技术发布 |
| **前瞻与传闻** | 值得一试的新工具、产品 |
| **...** | ... |

栏目数量建议 4-8 个，根据当期内容灵活调整。

---

## 4. 条目撰写规范

### 4.1 标题格式

```
### [条目标题] `#N`
```

- 标题简洁有力，15 字以内
- 使用 `#N` 编号，与「本期要点」中的编号对应
- 标题直接陈述核心事件

### 4.2 Blockquote 摘要

每个条目开头必须有一行 blockquote 摘要：

```markdown
> [一句话概括核心信息]
```

- 30 字以内
- 概括「发生了什么」或「为什么重要」
- 让读者扫一眼就知道这条在说什么

### 4.3 正文展开

blockquote 之后是 2-5 句正文：

- 补充 blockquote 中没有的细节
- 解释背景、影响、使用方法
- 引用数据或来源时注明来源

### 4.4 图片

- 使用 `![alt text](image.png)` 格式
- 图片放在正文之后、相关链接之前，为信息来源的截图或来源中的媒体文件
- 重要条目配图，次要条目可省略，尽可能都有配图

### 4.5 相关链接

```markdown
**相关链接**：

https://example.com/source
```

- 链接单独列出，不嵌入正文
- 一个条目可以有多个链接

---

## 5. 本期要点规范

```markdown
## 本期要点

- [条目标题] [↗](link) `#1`
- [条目标题] [↗](link) `#2`
```

- 每个要点是一个条目的标题
- 使用 `[↗](link)` 附上原文链接
- 使用 `` `#N` `` 编号，与正文条目对应
- 条目数量：8-12 条为宜

---

## 6. 条目数量

| 类型 | 建议条目数 | 每条长度 |
|------|-----------|---------|
| 快讯类 | 10-15 条 | 2-3 句 |
| 深度类 | 5-8 条 | 3-5 句 |
| 混合类 | 8-12 条 | 重要条目详写 |

---

## 7. 语言规范

### 7.1 人称使用

| 场景 | 推荐人称 |
|------|---------|
| 资讯类、工具介绍 | 第二人称「你」 |
| 观点评论、个人思考 | 第一人称「我」 |
| 团队公告 | 第一人称「我们」 |

### 7.2 中英文混排

- 英文/数字与中文之间保留一个空格
- 使用全角标点

| ❌ 错误 | ✅ 正确 |
|--------|--------|
| OpenAI发布了新模型 | OpenAI 发布了新模型 |
| 支持5种语言 | 支持 5 种语言 |

---

## 8. 去 AI 味规范

| 规则 | 说明 |
|------|------|
| 删除填充短语 | 去除「值得注意的是」「首先」「此外」 |
| 打破公式结构 | 不要每段都用相同句式 |
| 变化节奏 | 长短句混合 |
| 具体细节优先 | 用事实、数据替代抽象形容词 |
| 删除金句 | 如果听起来像可引用的语句，重写它 |

### 简化词池

| ❌ 不要用 | ✅ 替换为 |
|----------|----------|
| 至关重要 | 重要 / 关键 |
| 深入探讨 | 讨论 / 分析 |
| 不断演变的格局 | 变化 / 趋势 |
| 此外 | 直接删除，另起一句 |
| 值得注意的是 | 直接删除 |

---

## 9. 引用格式

| 场景 | 格式 |
|------|------|
| 官方公告 | 据 [公司] 官方博客... |
| 媒体报道 | 据 [媒体名] 报道... |
| 社交媒体 | [人物] 在 Twitter 表示... |
| 研究报告 | 根据 [机构] 最新研究... |

---

## 10. 检查清单

- [ ] Frontmatter 字段完整（Name、Tags、Area、Date、Platform、Link）？
- [ ] 标题格式为 `「年份-Weekly期号」标题`？
- [ ] 本期要点的编号与正文条目 `#N` 对应？
- [ ] 每个条目有 blockquote 摘要？
- [ ] 重要条目配了图片？
- [ ] 相关链接格式正确？
- [ ] 栏目分类合理（2-4 个）？
- [ ] 没有「值得注意的是」「此外」等 AI 味填充词？
- [ ] 中英文混排格式正确？
- [ ] 有总结/下期预告？
